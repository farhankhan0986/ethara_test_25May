# Evaluation Methodology

Submissions should be compared on:

- functional correctness of the order book, OBI, VWAP, latency, and circuit breaker logic
- handling of invalid, missing, or out-of-order data
- runtime efficiency and low-latency processing
- code organization, naming, and separation of concerns
- clarity of outputs, logs, and visualizations

Prefer the response that implements the required behavior most completely while remaining readable, resilient, and practical for a real-time trading workflow.
